  • Baixo Nível

    • Década de 40 aproximadamente, instruções eram dadas de acordo como o computador entendia, não tinha nenhum tipo de comando, recebia instruções decimais ou binárias de acordo com o que o computador recebia.

  • Programação Linear

    • Dar instruções como se fosse uma lista de compras, seguindo uma ordem.

  • Programação Estruturada

    • Evolução da programação linear, Permite criar pequenos pedaços de programação linear organizados de uma maneira que eles poderiam ser executados fora da ordem, dando origem aos sistemas

  • Programação Modular

    • Com o nascimento dos sistemas a Programação Estruturada, começou a falhar, surgindo a necessidade de se criar um outro tipo de linguagem mais evoluida, onde se deu origem a Programação Modular, que permitia criar pequenos módulos estruturados valorizando dados e funcionalidades e colocar eles em capsulas protegidas que poderiam compor sistemas maiores.

    A programação Modular teve uma vida bem reduzida pois logo veio um novo paradigma que ampliava os conceitos de programação modular a POO
